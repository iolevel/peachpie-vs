Getting Started

You've just created a basic ASP.NET Core application which in addition serves compiled PHP scripts.

The solution consists of two separate projects:

  • $safeprojectname$; is the PHP project. Its PHP script files (*.php) get compiled and served by the web server.
  • $safeprojectname$.Server; implements the default Web Server. It is a startup project which handles web requests and serves the compiled PHP scripts, static files, Razor views and other ASP.NET Core end-points.

PHP Source Files

PHP source files are placed within the $safeprojectname$ project. Upon building the project, the compiler parses the files, provides additional diagnostics, and outputs a single binary .dll file.

Content Files

Unlike in regular PHP application, compile and content files are treated separately here. Content files are not compiled and they will only be served by the web server as a static file.

Place your content files (images and stylesheets, and other non-compile items) into the /wwwroot/ folder within the $safeprojectname$.Server project.
